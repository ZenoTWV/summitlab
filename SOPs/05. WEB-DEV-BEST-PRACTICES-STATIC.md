## 1. Visual Design & Hierarchy

### 1.1 Visual Hierarchy Fundamentals

#### Core Principle
Visual hierarchy determines what users see first, second, and third. It's the single most important design element for guiding user attention and ensuring critical information is consumed.

**The F-Pattern Myth:**
- ❌ Don't: Design assuming users read in strict F-patterns
- ✅ Do: Use size, contrast, and spacing to create clear hierarchy regardless of reading pattern

**Hierarchy Through Scale:**
Make the most important elements the biggest and boldest. For each subsequent element of decreasing importance, reduce size, weight, or contrast.

**Implementation:**
```css
/* Clear hierarchy example */
.hero-title {
  font-size: 64px;
  font-weight: 700;
  line-height: 1.1;
  margin-bottom: 24px;
}

.hero-subtitle {
  font-size: 24px;
  font-weight: 400;
  line-height: 1.5;
  opacity: 0.8;
}

.hero-cta {
  font-size: 18px;
  font-weight: 600;
  /* High contrast for primary action */
}
```

**Common Mistakes:**
- ❌ Equal weight columns splitting attention (no clear focal point)
- ❌ Too many elements competing for attention
- ❌ Tiny text that requires effort to read
- ✅ Single dominant element per section
- ✅ Clear progression from primary to secondary to tertiary elements

### 1.2 Typography

#### Font Selection

**General Rules:**
- Limit to 2 font families maximum (one for headings, one for body)
- Use web-safe fonts for consistency across browsers
- Choose fonts that match your brand positioning:
  - **Geometric sans-serifs:** Modern, technical, clean (e.g., Linear, SaaS products)
  - **Serifs:** Traditional, trustworthy, established (e.g., financial services)
  - **Handwritten/Custom:** Organic, personal, creative (e.g., artisan products)

**Font Pairing:**
```css
/* Example: Modern SaaS */
h1, h2, h3, h4, h5, h6 {
  font-family: 'Inter', -apple-system, system-ui, sans-serif;
  font-weight: 700;
}

body, p {
  font-family: 'Inter', -apple-system, system-ui, sans-serif;
  font-weight: 400;
}
```

#### Font Sizing and Scale

**Critical Rules:**
- **Body text minimum:** 16px (never below - triggers iOS zoom)
- **Line height:** 1.5-1.6 for body text, 1.1-1.3 for headings
- **Line length:** 50-75 characters per line for optimal readability

**Responsive Typography Scale:**
```css
/* Desktop scale */
h1 { font-size: 64px; }
h2 { font-size: 48px; }
h3 { font-size: 32px; }
h4 { font-size: 24px; }
body { font-size: 18px; }

/* Mobile scale */
@media (max-width: 768px) {
  h1 { font-size: 40px; }
  h2 { font-size: 32px; }
  h3 { font-size: 24px; }
  h4 { font-size: 20px; }
  body { font-size: 16px; }
}
```

**Common Mistakes:**
- ❌ Font size below 16px for body text
- ❌ Mixing title case and sentence case inconsistently
- ❌ Line height too tight (causes readability issues)
- ❌ Using decorative fonts for paragraph text
- ✅ Consistent casing throughout site
- ✅ Adequate spacing between lines
- ✅ Simple, readable fonts for body content

#### Variable Fonts (2026 Trend)

Modern font families now offer variable weights, slants, and even animated transitions between states.

**Benefits:**
- Single file contains multiple weights (reduces HTTP requests)
- Smooth animations between font states
- Icon packages with built-in animations

**Usage:**
```css
@font-face {
  font-family: 'InterVariable';
  src: url('Inter-Variable.woff2') format('woff2');
  font-weight: 100 900; /* Variable range */
}

.dynamic-heading {
  font-family: 'InterVariable';
  font-weight: 400;
  transition: font-weight 0.3s ease;
}

.dynamic-heading:hover {
  font-weight: 700; /* Smooth transition */
}
```

### 1.3 Color Theory

#### Color Palette Structure

**60/30/10 Rule:**
- **60%:** Dominant colors (grays, blacks, whites, backgrounds)
- **30%:** Brand colors (blues, greens, company identity)
- **10%:** Accent colors (CTAs, highlights, urgent elements)

**Practical Application:**
```css
:root {
  /* Dominant (60%) */
  --color-bg-primary: #ffffff;
  --color-bg-secondary: #f5f5f5;
  --color-text-primary: #1a1a1a;
  --color-text-secondary: #666666;
  
  /* Brand (30%) */
  --color-brand-primary: #0066cc;
  --color-brand-secondary: #004c99;
  
  /* Accent (10%) */
  --color-accent-cta: #ff6b35;
  --color-accent-success: #28a745;
}
```

#### Accessibility & Contrast

**WCAG 2.2 AA Standards:**
- **Normal text (< 18px):** Minimum 4.5:1 contrast ratio
- **Large text (≥ 18px or ≥ 14px bold):** Minimum 3:1 contrast ratio
- **UI components:** Minimum 3:1 contrast ratio

**Testing Tool:**
Use coolers.co or WebAIM Contrast Checker to validate color combinations.

**Common Mistakes:**
- ❌ Light gray text on white backgrounds (poor contrast)
- ❌ Yellow buttons (hard to read white text on yellow)
- ❌ Relying solely on color to convey information
- ✅ Test all color combinations with contrast checker
- ✅ Provide alternative indicators beyond color (icons, text, patterns)

**Example - Accessible Button:**
```css
/* Bad: Poor contrast */
.button-bad {
  background-color: #ffeb3b; /* Yellow */
  color: #ffffff; /* White text - fails contrast */
}

/* Good: High contrast */
.button-good {
  background-color: #0066cc; /* Blue */
  color: #ffffff; /* White text - passes contrast */
}
```

#### Color Psychology for Brands

**Strategic Color Choice:**
- **Blue:** Trust, stability, technology (Stripe, Coinbase)
- **Green:** Growth, organic, eco-friendly (sustainability brands)
- **Purple:** Innovation, creativity, AI (common in AI startups - "AI slop")
- **Red/Orange:** Energy, urgency, action (CTAs, sales)
- **Black/White:** Minimalism, luxury, sophistication (Apple, luxury brands)

**Avoid "AI Slop" Aesthetics:**
Purple gradients, excessive shadows, and generic color schemes are overused in AI-generated designs. Differentiate by:
- Using muted, sophisticated color palettes
- Limiting gradients to subtle backgrounds
- Choosing unexpected but brand-appropriate colors

### 1.4 Spacing and Layout

#### Spacing Fundamentals

**Why Spacing Matters:**
Proper spacing creates visual structure, improves readability, and signals quality. Poor spacing makes designs feel cramped, unprofessional, or confusing.

**8-Point Grid System:**
Use multiples of 8px for all spacing and sizing to maintain consistency.

```css
:root {
  --space-xs: 8px;
  --space-sm: 16px;
  --space-md: 24px;
  --space-lg: 32px;
  --space-xl: 48px;
  --space-xxl: 64px;
}

.section {
  padding: var(--space-xl) var(--space-lg);
  margin-bottom: var(--space-xxl);
}
```

**Common Spacing Mistakes:**
- ❌ Inconsistent spacing (14px here, 18px there)
- ❌ Too much space between related elements
- ❌ Too little space between unrelated sections
- ❌ Text touching edges of containers
- ✅ Related elements grouped with less space
- ✅ Unrelated sections separated with more space
- ✅ Consistent padding around all containers

#### Layout Systems

**Modular/Grid Layouts (2026 Trend):**
CSS Grid enables complex, structured layouts that adapt to content.

```css
/* Bento Grid Layout */
.bento-grid {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(300px, 1fr));
  gap: 24px;
  padding: 48px 24px;
}

.bento-item {
  background: var(--color-bg-secondary);
  border-radius: 12px;
  padding: 32px;
  transition: transform 0.3s ease;
}

.bento-item:hover {
  transform: translateY(-4px);
}
```

**Responsive Breakpoints:**
```css
/* Mobile first approach */
.container {
  width: 100%;
  padding: 16px;
}

/* Tablet */
@media (min-width: 768px) {
  .container {
    padding: 24px;
    max-width: 720px;
    margin: 0 auto;
  }
}

/* Desktop */
@media (min-width: 1200px) {
  .container {
    padding: 48px;
    max-width: 1140px;
  }
}
```

## 2. User Experience (UX)

### 2.1 Navigation Design

#### Navigation Placement

**Expected Locations:**
- **Top navigation:** Industry standard, always visible
- **Left sidebar:** Common for dashboards and web apps
- **Hamburger menu (mobile):** Acceptable on small screens only

**Critical Rules:**
- Logo must link to homepage
- Navigation must be consistent across all pages
- Current page should be visually indicated
- Maximum 7 top-level items (avoid cognitive overload)

**Implementation:**
```html
<nav class="main-nav" aria-label="Main navigation">
  <a href="/" class="logo">Company Name</a>
  <ul class="nav-links">
    <li><a href="/features" aria-current="page">Features</a></li>
    <li><a href="/pricing">Pricing</a></li>
    <li><a href="/about">About</a></li>
    <li><a href="/contact">Contact</a></li>
  </ul>
  <div class="nav-cta">
    <a href="/signup" class="button-primary">Get Started</a>
  </div>
</nav>
```

```css
.main-nav {
  position: sticky;
  top: 0;
  z-index: 100;
  background: var(--color-bg-primary);
  padding: 16px 24px;
  display: flex;
  justify-content: space-between;
  align-items: center;
}

.nav-links a[aria-current="page"] {
  font-weight: 600;
  color: var(--color-brand-primary);
  border-bottom: 2px solid var(--color-brand-primary);
}
```

**Common Mistakes:**
- ❌ Navigation that appears/disappears unexpectedly
- ❌ Too many navigation items (overwhelming)
- ❌ No visual indicator of current page
- ❌ Logo that doesn't link to homepage
- ✅ Sticky navigation for easy access
- ✅ Clear visual hierarchy in menu items
- ✅ Breadcrumbs for complex sites

#### Breadcrumbs

For sites with deep hierarchies, breadcrumbs improve navigation:

```html
<nav aria-label="Breadcrumb">
  <ol class="breadcrumb">
    <li><a href="/">Home</a></li>
    <li><a href="/products">Products</a></li>
    <li aria-current="page">Product Name</li>
  </ol>
</nav>
```

### 2.2 Above-the-Fold Optimization

#### Hero Section Essentials

The hero section (first screen) must answer three questions in 5 seconds:
1. **What is this?** (Clear value proposition)
2. **Is it for me?** (Target audience identification)
3. **What should I do next?** (Clear primary CTA)

**Critical Elements:**
- **Headline (H1):** Specific, benefit-focused, not clever/vague
- **Subheadline:** Expands on headline, adds crucial context
- **Primary CTA:** High contrast, action-oriented copy
- **Visual:** Product screenshot, demo, or relevant imagery
- **Trust signals:** Logos, user count, testimonials

**Example Structure:**
```html
<section class="hero">
  <div class="hero-content">
    <!-- Clear, specific headline -->
    <h1>AI-Powered Issue Tracking for Fast-Moving Teams</h1>
    
    <!-- Subheadline with context -->
    <p class="hero-subtitle">
      Linear helps startups ship faster with the most responsive
      issue tracker on the market.
    </p>
    
    <!-- Clear, contrasting CTA -->
    <div class="hero-cta">
      <a href="/signup" class="button-primary">Start Building — Free</a>
      <a href="/demo" class="button-secondary">Watch Demo</a>
    </div>
    
    <!-- Trust signals -->
    <div class="social-proof">
      <p>Trusted by fast-moving teams at</p>
      <div class="logo-grid">
        <!-- Company logos -->
      </div>
    </div>
  </div>
  
  <div class="hero-visual">
    <!-- Product screenshot or demo -->
    <img src="/product-ui.png" alt="Linear dashboard interface">
  </div>
</section>
```

**Common Mistakes:**
- ❌ Vague headlines ("The Future of Work")
- ❌ Clever wordplay that obscures meaning
- ❌ No clear CTA above fold
- ❌ Generic stock photos unrelated to product
- ❌ Autoplay video with sound
- ✅ Specific, benefit-driven headline
- ✅ Product screenshot showing actual UI
- ✅ Single dominant CTA
- ✅ Clear target audience identification

#### The 5-Second Test

**What Users Should Know in 5 Seconds:**
- Company/product name
- What the product does (specifically)
- Who it's for
- Primary action to take

**Testing Your Hero:**
Show your homepage to someone for 5 seconds, then ask:
1. What does this company do?
2. Who is it for?
3. What would you click next?

If they can't answer all three, revise your hero section.

### 2.3 Mobile-First Approach

#### Why Mobile-First Matters

**Current Reality (2026):**
- 60%+ of traffic is mobile
- Google uses mobile-first indexing (ranks based on mobile version)
- Core Web Vitals measured primarily on mobile
- iOS Safari triggers zoom on text < 16px

**Mobile-First Design Process:**
1. Design for smallest screen first (320px width)
2. Add complexity for larger screens (progressive enhancement)
3. Test on real devices, not just browser dev tools

**Touch Target Sizing:**
```css
/* Minimum touch target: 44x44px */
.button,
.link,
.form-input {
  min-height: 44px;
  min-width: 44px;
  padding: 12px 24px;
}

/* Adequate spacing between touch targets */
.button-group .button {
  margin: 8px;
}
```

**Mobile-Optimized Typography:**
```css
/* Mobile base */
body {
  font-size: 16px; /* Never below this */
  line-height: 1.6;
}

/* Scale up for desktop */
@media (min-width: 1200px) {
  body {
    font-size: 18px;
  }
}
```

**Common Mobile Mistakes:**
- ❌ Text smaller than 16px (triggers zoom)
- ❌ Touch targets smaller than 44x44px
- ❌ Horizontal scrolling required
- ❌ Forms with tiny input fields
- ❌ Popups that cover entire mobile screen
- ✅ Large, tappable buttons
- ✅ Adequate spacing between interactive elements
- ✅ Thumb-friendly navigation
- ✅ Forms optimized for mobile input

### 2.4 Scannability

#### Users Don't Read, They Scan

**Reality Check:**
Users scan headlines and bold text to decide if content is relevant. They only read body text if intrigued by headlines.

**Headline-Only Test:**
Read only your H1, H2, H3 tags in order. Do they tell a complete story? Can users understand your offering from headlines alone?

**Bad Example:**
```
H1: Welcome
H2: Our Mission
H3: What We Do
H4: Get Started
```
These headlines tell you nothing about the product.

**Good Example:**
```
H1: AI-Powered Sales Outreach for Agencies
H2: Automate Lead Generation in Minutes
H3: Personalized Campaigns at Scale
H4: Start Your Free Trial Today
```
These headlines tell the complete story.

**Formatting for Scannability:**
- Use short paragraphs (2-3 sentences maximum)
- Bullet points for lists (when appropriate)
- Bold key terms and benefits (sparingly)
- Adequate white space between sections
- Clear visual breaks between topics

**Common Mistakes:**
- ❌ Walls of text with no breaks
- ❌ Generic headlines that could apply to any product
- ❌ Burying key information in paragraphs
- ❌ No visual hierarchy in content
- ✅ Clear, descriptive headlines
- ✅ Short paragraphs with breathing room
- ✅ Key benefits highlighted early
- ✅ Scannable structure throughout

## 3. Performance Optimization

### 3.1 Core Web Vitals (2025/2026 Standards)

#### Current Thresholds

**LCP (Largest Contentful Paint):**
- ✅ Good: < 2.5s
- ⚠️ Needs Improvement: 2.5s - 4.0s
- ❌ Poor: > 4.0s
- **Target for competitive niches:** < 2.0s

**INP (Interaction to Next Paint):**
- ✅ Good: < 200ms
- ⚠️ Needs Improvement: 200ms - 500ms
- ❌ Poor: > 500ms
- **2026 Target:** < 150ms
- **Note:** INP replaced FID (First Input Delay) in 2024

**CLS (Cumulative Layout Shift):**
- ✅ Good: < 0.1
- ⚠️ Needs Improvement: 0.1 - 0.25
- ❌ Poor: > 0.25
- **Target:** < 0.08

**Why This Matters:**
- Direct Google ranking factor
- 75% of users must experience "good" metrics
- Mobile performance more critical than desktop
- Real User Monitoring (RUM) data > lab tests

### 3.2 Image Optimization

#### Modern Image Formats

**Format Priority:**
1. **AVIF** (best compression, newest)
2. **WebP** (excellent compression, wide support)
3. **JPEG/PNG** (fallback)

**Implementation:**
```html
<picture>
  <source srcset="hero-image.avif" type="image/avif">
  <source srcset="hero-image.webp" type="image/webp">
  <img src="hero-image.jpg" alt="Product dashboard" loading="lazy">
</picture>
```

#### Responsive Images

```html
<!-- Different sizes for different viewports -->
<img 
  srcset="
    image-320w.jpg 320w,
    image-640w.jpg 640w,
    image-1024w.jpg 1024w,
    image-1920w.jpg 1920w
  "
  sizes="
    (max-width: 640px) 100vw,
    (max-width: 1024px) 50vw,
    33vw
  "
  src="image-1024w.jpg"
  alt="Descriptive alt text"
  loading="lazy"
>
```

#### Lazy Loading

```html
<!-- Native lazy loading -->
<img src="image.jpg" loading="lazy" alt="Description">

<!-- Eager loading for LCP images -->
<img src="hero.jpg" loading="eager" fetchpriority="high" alt="Hero image">
```

**Critical Rules:**
- Use `loading="eager"` and `fetchpriority="high"` for LCP elements
- Use `loading="lazy"` for below-fold images
- Always compress images before upload
- Descriptive filenames (good for SEO)
- Always include alt text

### 3.3 Code Optimization

#### Resource Loading

**Preload Critical Resources:**
```html
<head>
  <!-- Preload critical fonts -->
  <link rel="preload" href="/fonts/inter-var.woff2" as="font" type="font/woff2" crossorigin>
  
  <!-- Preload critical CSS -->
  <link rel="preload" href="/css/critical.css" as="style">
  
  <!-- Preconnect to external domains -->
  <link rel="preconnect" href="https://fonts.googleapis.com">
  <link rel="preconnect" href="https://cdn.example.com">
</head>
```

#### Minification & Compression

**CSS/JS Minification:**
- Remove whitespace, comments, unused code
- Combine files where possible
- Use build tools (Vite, Webpack, Rollup)

**Server Compression:**
```nginx
# Enable Gzip/Brotli compression
gzip on;
gzip_types text/css application/javascript image/svg+xml;
gzip_min_length 1000;
```

#### Preventing Layout Shift (CLS)

**Reserve Space for Dynamic Content:**
```css
/* Reserve space for ad units */
.ad-container {
  width: 300px;
  height: 250px;
  background: #f0f0f0; /* Placeholder while loading */
}

/* Reserve space for images */
.image-container {
  aspect-ratio: 16 / 9; /* Prevents layout shift */
}
```

**Avoid Common CLS Triggers:**
- ❌ Images without dimensions
- ❌ Ads inserted without reserved space
- ❌ Dynamically injected content
- ❌ Web fonts causing text reflow
- ✅ Set explicit width/height or aspect-ratio
- ✅ Reserve space for ads/dynamic content
- ✅ Use font-display: swap carefully
- ✅ Avoid inserting content above existing content

## 4. SEO Fundamentals

- ✅ Single H1 per page
- ✅ Logical heading hierarchy
- ✅ Descriptive, keyword-rich titles
- ✅ Unique meta descriptions per page

### 4.2 Content Strategy

#### Topic Clusters Over Keywords

**Modern SEO Philosophy (2025/2026):**
Build authority through comprehensive topic coverage, not keyword stuffing.

**Content Structure:**
- **Hub Pages:** Comprehensive guides (2000-5000 words) covering entire topic
- **Cluster Pages:** Specific subtopics linking back to hub
- **Internal Linking:** Connect related content strategically

**Example Structure:**
```
Hub: "AI Sales Automation Guide"
├── Cluster: "Email Personalization with AI"
├── Cluster: "Lead Scoring Algorithms"
├── Cluster: "Outreach Sequence Optimization"
└── Cluster: "CRM Integration Best Practices"
```

**Content Quality Standards:**
- Focus on E-E-A-T: Experience, Expertise, Authoritativeness, Trustworthiness
- Update existing content > creating new content
- Target 50-200 high-quality pages (not 1000s of thin pages)
- Answer questions in 40-60 words for featured snippets
- Focus on 4-6 topical pillars maximum

#### Content Types by Intent

**Informational:**
- Blog posts, guides, tutorials
- "How to," "What is," "Best practices"
- Goal: Build authority, attract top-of-funnel traffic

**Commercial:**
- Product comparisons, reviews, case studies
- "Best X for Y," "[Product] vs [Competitor]"
- Goal: Capture consideration-stage users

**Transactional:**
- Product pages, pricing, signup flows
- "Buy," "Get started," "Sign up"
- Goal: Convert ready buyers

**Local:**
- Location pages, service areas
- "[Service] in [City]"
- Goal: Capture local searches

### 4.3 Technical SEO

#### URL Structure

**Clean, Descriptive URLs:**
```
✅ Good: example.com/blog/ai-outreach-guide
❌ Bad: example.com/p?id=12345&cat=blog
```

**Rules:**
- Use hyphens (not underscores)
- Keep URLs short and descriptive
- Include target keyword
- Avoid special characters
- Use lowercase only

#### XML Sitemap

```xml

<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
```

### 5.2 POUR Principles

#### Perceivable

**Text Alternatives:**
```html
<!-- Images need alt text -->
<img src="product.jpg" alt="OutreachX dashboard showing email sequences">

<!-- Decorative images use empty alt -->
<img src="decorative-line.svg" alt="">

<!-- Complex images need longer descriptions -->
<figure>
  <img src="chart.png" alt="Bar chart showing 300% increase in meetings booked">
  <figcaption>
    Detailed description: Chart shows meetings booked per month...
  </figcaption>
</figure>
```

**Color Contrast:**
- Minimum 4.5:1 for normal text
- Minimum 3:1 for large text (18px+ or 14px+ bold)
- Minimum 3:1 for UI components
- Don't rely on color alone to convey information

**Example:**
```html
<!-- Bad: Color only -->
<p style="color: red;">Error: Form invalid</p>

<!-- Good: Color + icon + text -->
<p class="error">
  <svg aria-hidden="true"><!-- Error icon --></svg>
  <span class="sr-only">Error:</span>
  Form invalid - please check required fields
</p>
```

#### Operable

**Keyboard Navigation:**
All interactive elements must be keyboard accessible.

```html
<!-- Buttons are naturally keyboard accessible -->
<button onclick="submitForm()">Submit</button>

<!-- DIVs as buttons need extra work -->
<div 
  role="button" 
  tabindex="0" 
  onclick="submitForm()"
  onkeypress="handleKeyPress(event)">
  Submit
</div>

<!-- Better: Just use a button -->
<button>Submit</button>
```

**Focus Indicators:**
```css
/* Visible focus state */
button:focus,
a:focus {
  outline: 2px solid var(--color-brand-primary);
  outline-offset: 2px;
}

/* Never remove outline without replacement */
/* ❌ Bad */
*:focus {
  outline: none; /* Terrible for accessibility */
}
```

**Skip Navigation:**
```html
<!-- Allows keyboard users to skip to main content -->
<a href="#main-content" class="skip-link">Skip to main content</a>

<main id="main-content">
  <!-- Page content -->
</main>
```

```css
.skip-link {
  position: absolute;
  top: -40px;
  left: 0;
  background: var(--color-brand-primary);
  color: white;
  padding: 8px;
  z-index: 100;
}

.skip-link:focus {
  top: 0;
}
```

**No Seizure Triggers:**
- No flashing content more than 3 times per second
- Avoid rapid animations or strobing effects

#### Understandable

**Clear Language:**
- Use simple, plain language
- Explain jargon and acronyms
- Logical reading order
- Consistent navigation across pages

**Form Labels:**
```html
<!-- Clear labels for all inputs -->
<label for="email">Email Address</label>
<input type="email" id="email" name="email" required>

<!-- Helpful error messages -->
<input 
  type="email" 
  id="email" 
  required
  aria-describedby="email-error">
<span id="email-error" class="error" role="alert">
  Please enter a valid email address (e.g., name@example.com)
</span>
```

**Descriptive Link Text:**
```html
<!-- ❌ Bad: Generic link text -->
<p>Click <a href="/features">here</a> to learn more.</p>

<!-- ✅ Good: Descriptive text -->
<p><a href="/features">Learn more about our features</a>.</p>
```

#### Robust

**Semantic HTML:**
```html
<!-- Use proper HTML5 elements -->
<header>
  <nav aria-label="Main navigation">
    <!-- Navigation links -->
  </nav>
</header>

<main>
  <article>
    <h1>Article Title</h1>

  </article>
</main>

<aside aria-label="Related content">
  <!-- Sidebar -->
</aside>

<footer>
  <!-- Footer content -->
</footer>
```

**ARIA Labels:**
```html
<!-- Button with icon only -->
<button aria-label="Close menu">
  <svg aria-hidden="true"><!-- X icon --></svg>
</button>

<!-- Region labels -->
<section aria-labelledby="features-heading">
  <h2 id="features-heading">Key Features</h2>
  <!-- Features -->
</section>

<!-- Live regions for dynamic content -->
<div role="status" aria-live="polite">

</div>
```

**Valid HTML:**
- No unclosed tags
- Proper nesting
- Unique IDs
- Correct attribute usage

**Three Levels:**
1. **Primary CTA:** Most important action (sign up, buy now)
2. **Secondary CTA:** Alternative action (learn more, watch demo)
3. **Tertiary CTA:** Low-commitment options (read docs, contact sales)

```css
  color: var(--color-brand-primary);
  border: 2px solid var(--color-brand-primary);
```


**Placement Rules:**
- Primary CTA above the fold
- Primary CTA visible without scrolling
- Repeat CTA at logical conversion points (end of features section, after testimonials)

**Common Mistakes:**
- ❌ Multiple equally-weighted CTAs (creates decision paralysis)

- ❌ No CTA above the fold
- ✅ Clear visual hierarchy (primary > secondary > tertiary)

## 8. AI-Assisted Development (Vibe Coding)

### 8.1 Introduction to Vibe Coding

**What is Vibe Coding?**
Vibe coding (or vibe design) is the practice of using AI tools (like Cursor, Claude Code, Windsurf, Lovable, v0, Replit) to rapidly build and iterate on websites and applications through natural language prompts rather than writing every line of code manually.

**Key Benefits:**
- Compress development timelines (weeks → hours)
- Iterate in real-time with clients
- Focus on design and UX rather than syntax
- Lower barrier to entry for non-developers

**Common Tools:**
- **Cursor:** AI-first code editor (best for implementation)
- **Windsurf:** AI coding assistant (good for long-term planning)
- **Claude Code:** Command-line agentic coding
- **Lovable/v0/Replit:** Visual AI website builders
- **ChatGPT/Claude:** General AI assistants

#### Use Git Religiously

**Critical Practice:**
Always commit working code before trying to add new features.

```bash
# After completing hero section
git add .
git commit -m "Add hero section with CTAs"

# Now safe to work on next section
# If AI breaks something, can easily revert
git reset --hard HEAD
```

**Why This Matters:**
AI often makes unnecessary changes to working code while implementing new features. Git allows you to revert these regressions and start fresh.

---

#### When AI Gets Stuck

**Signs AI Is Struggling:**
- Multiple failed attempts at same bug
- Adding layers of complicated code
- Making changes to unrelated files
- Going in circles

**Solutions:**
1. **Reset and try different approach:**
   ```bash
   git reset --hard HEAD
   ```
   Then prompt with different strategy

2. **Switch AI models:**
   Try Claude 3.7 Sonnet vs GPT-4 vs Gemini
   Different models excel at different tasks

3. **Ask AI to explain the problem:**
   ```
   Let's step back. Before fixing this, explain what you think
   is causing the bug and three possible approaches to fix it.
   ```

4. **Simplify the request:**
   ```
   Let's forget the animation for now. Just get the basic
   functionality working first.
   ```

5. **Provide more context:**
   ```
   Here's the error message from the console:
   [Paste error]
   
   Here's the relevant code:
   [Paste code]
   
   Here's what I expected to happen:
   [Describe expected behavior]
   ```

### 8.4 Testing AI-Generated Code

#### Write Tests First

**AI is good at writing tests:**
```
Write integration tests for this signup form:

Requirements:
- Test that form validates email format
- Test that form shows error for missing fields
- Test that form submits successfully with valid data
- Test that success message appears after submission
```

**Use tests to catch regressions:**
When AI makes changes, tests will catch if it breaks existing functionality.

---

#### Manual Testing Checklist

**Every time AI generates/modifies code:**
- [ ] Test on mobile (real device, not just browser dev tools)
- [ ] Test all interactive elements (buttons, forms, links)
- [ ] Check console for errors
- [ ] Validate HTML (W3C validator)
- [ ] Check accessibility (WAVE, axe DevTools)
- [ ] Test keyboard navigation
- [ ] Verify responsive breakpoints
- [ ] Check page load performance (Lighthouse)

### 8.5 Common AI Mistakes

**What AI Often Gets Wrong:**
- Inconsistent styling (mixes random font sizes, spacing)
- Generic "AI slop" aesthetics (purple gradients, excessive shadows)
- Breaks responsive design when adding features
- Makes unnecessary changes to working code
- Uses deprecated or incorrect syntax
- Overcomplicates simple tasks

**How to Prevent:**
- Provide explicit style guide
- Reference specific design systems
- Review code before accepting changes
- Use Git to catch regressions
- Test thoroughly after each change

---

### 8.6 When to Override AI

**AI is a tool, not a replacement for judgment:**

**Override when:**
- AI suggests approach that conflicts with best practices
- Generated code has accessibility issues
- Design doesn't match brand guidelines
- Performance suffers from AI's implementation
- Code is unnecessarily complex

**Example:**
```
# AI suggests:
<div onclick="handleClick()">Click me</div>

# You should change to:
<button onclick="handleClick()">Click me</button>
```

**Trust your expertise:**
Just because AI generated it doesn't mean it's optimal. Review critically.

### 9.2 Playful/Interactive Design

**Characteristics:**
- Mouse-following elements
- Scroll-triggered animations
- Interactive hover states
- Micro-interactions

**Example:**
```javascript
// Mouse-following element
document.addEventListener('mousemove', (e) => {
  const cursor = document.querySelector('.custom-cursor');
  cursor.style.left = e.clientX + 'px';
  cursor.style.top = e.clientY + 'px';
});

// Scroll-triggered animation
const observer = new IntersectionObserver((entries) => {
  entries.forEach(entry => {
    if (entry.isIntersecting) {
      entry.target.classList.add('animate-in');
    }
  });
});

document.querySelectorAll('.animate-on-scroll').forEach(el => {
  observer.observe(el);
});
```

**Use sparingly:**
Too much interactivity can be distracting. Reserve for key moments.

### 9.4 Gradient Backgrounds

**Trend:**
Beautiful, subtle gradients replacing solid backgrounds.

**Implementation:**

```css
/* Subtle gradient */
.hero {
```

### 9.8 Variable Fonts

**2026 Feature:**
Single font file with multiple weights/styles.

**Benefits:**

- Animated typography effects

**Implementation:**

```css
  animation: weight-pulse 3s ease infinite;
}

@keyframes weight-pulse {
  0%, 100% { font-weight: 400; }
  50% { font-weight: 700; }
}
```

## 10. Common Pitfalls

### 10.1 Design Mistakes

#### Overcrowding

**Problem:**
Too many elements competing for attention creates visual chaos.

**Solution:**
- One dominant element per section
- Generous white space
- Clear visual hierarchy
- Remove unnecessary elements

**Example:**
```
❌ Bad Hero Section:
- 3 equal-sized headlines
- 5 different CTA buttons
- 10 features listed
- Customer logos
- Testimonial
- Video
- Pricing table

✅ Good Hero Section:
- 1 clear headline
- 1 supporting subheadline
- 1 primary CTA
- 1 secondary CTA
- Simple visual
```

---

#### Poor Color Contrast

**Problem:**
Low contrast makes content hard to read, fails accessibility.

**Solution:**
- Test all color combinations with contrast checker
- Minimum 4.5:1 for normal text
- Avoid light gray on white
- Avoid yellow buttons with white text

---

#### Inconsistent Spacing

**Problem:**
Random spacing values make design feel unprofessional.

**Solution:**
Use 8-point grid system consistently:
```css
--space-1: 8px;
--space-2: 16px;
--space-3: 24px;
--space-4: 32px;
--space-5: 40px;
--space-6: 48px;
```

Never use random values like 13px, 27px, etc.

---

#### Too Many Font Families

**Problem:**
Mixing many fonts creates visual discord.

**Solution:**
- Maximum 2 font families
- One for headings, one for body
- Or single variable font for both

---

### 10.2 Development Mistakes

#### Not Using Semantic HTML

**Problem:**
```html
<div class="header">
  <div class="nav">
    <div class="nav-item">Home</div>
  </div>
</div>
```

**Solution:**
```html
<header>
  <nav aria-label="Main navigation">
    <a href="/">Home</a>
  </nav>
</header>
```

---

#### Forgetting Mobile

**Problem:**
Designing for desktop only, treating mobile as afterthought.

**Solution:**
- Design mobile-first
- Test on real devices
- Touch targets minimum 44x44px
- Text minimum 16px

---

#### Not Testing Accessibility

**Problem:**
Skipping keyboard navigation, screen reader testing.

**Solution:**
- Test with keyboard only (no mouse)
- Test with screen reader
- Run automated tools (WAVE, axe)
- Check color contrast

---

### 10.3 Performance Mistakes

#### Unoptimized Images

**Problem:**
Uploading 5MB photos straight from camera.

**Solution:**
- Compress before upload (TinyPNG, Squoosh)
- Use modern formats (AVIF, WebP)
- Implement lazy loading
- Use responsive images (srcset)

---

#### Render-Blocking Resources

**Problem:**
```html
<head>
  <link rel="stylesheet" href="huge-framework.css">
  <script src="jquery.min.js"></script>
  <script src="bootstrap.min.js"></script>
</head>
```

**Solution:**
```html
<head>
  <!-- Inline critical CSS -->
  <style>/* Critical above-fold styles */</style>
  
  <!-- Defer non-critical CSS -->
  <link rel="stylesheet" href="styles.css" media="print" onload="this.media='all'">
  
  <!-- Defer/async scripts -->
  <script src="app.js" defer></script>
</head>
```

---

#### No Caching Strategy

**Problem:**
Every visit downloads all assets fresh.

**Solution:**
```nginx
# Cache static assets for 1 year
location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff2)$ {
  expires 1y;
  add_header Cache-Control "public, immutable";
}
```

---

### 10.4 SEO Mistakes

#### Generic Page Titles

**Problem:**
```html
<title>Home</title>
<title>About</title>
<title>Services</title>
```

**Solution:**
```html
<title>AI Sales Outreach Automation | OutreachX</title>
<title>About Our AI Agency | OutreachX</title>
<title>AI Implementation Services | OutreachX</title>
```

---

#### Missing Alt Text

**Problem:**
```html
<img src="product.jpg">
```

**Solution:**
```html
<img src="product.jpg" alt="OutreachX dashboard showing email campaign analytics">
```

---

#### No Schema Markup

**Problem:**
Missing structured data for rich results.

**Solution:**
Implement Organization, Product, FAQ, Article schemas (see SEO section).

---

### 10.5 Conversion Mistakes

#### Unclear Value Proposition

**Problem:**
```html
<h1>Welcome to Our Platform</h1>
<p>We help businesses grow.</p>
```

**Solution:**
```html
<h1>AI-Powered Sales Outreach for Agencies</h1>
<p>Book 3x more meetings with automated, personalized email campaigns</p>
```

---

#### Weak CTAs

**Problem:**
```html
<button>Submit</button>
<button>Learn More</button>
<button>Click Here</button>
```

**Solution:**
```html
<button>Start Free Trial</button>
<button>See How It Works</button>
<button>Book Strategy Call</button>
```

---

#### Too Many CTAs

**Problem:**
5 equally-weighted buttons competing for attention.

**Solution:**
- 1 primary CTA (high contrast)
- 1 secondary CTA (lower emphasis)
- Maybe 1 tertiary (text link)

## 13. Key Takeaways for 2025/2026

1. **Performance is Non-Negotiable**
   - Core Web Vitals directly impact rankings and conversions
   - Mobile performance more critical than desktop
   - 75% of users must experience "good" metrics

2. **Accessibility is Law**
   - April 2026 deadline approaching for WCAG 2.2 AA
   - Build accessibility in from the start (cheaper than retrofitting)
   - Automated tools catch 30-50% of issues—manual testing required

3. **Content Quality > Quantity**
   - Focus on 50-200 exceptional pages vs. thousands of mediocre ones
   - Topic authority beats keyword stuffing
   - Update existing content > creating new content

4. **Mobile-First Always**
   - 60%+ of traffic is mobile
   - Google uses mobile-first indexing
   - Design for smallest screen first, enhance for larger

5. **User Experience = SEO**
   - Google's algorithm prioritizes user satisfaction
   - Core Web Vitals are ranking factors
   - Dwell time, bounce rate, engagement matter

6. **AI is Everywhere**
   - 44% of queries show AI Overviews
   - Optimize for AI-generated summaries
   - E-E-A-T more important than ever

7. **Authenticity Wins**
   - Real photos > stock images
   - Genuine content > generic AI slop
   - Human connection > corporate polish
   - Avoid overused AI aesthetics (purple gradients, excessive shadows)

8. **Measurement Matters**
   - Use data to guide decisions, not assumptions
   - A/B test major changes
   - Real User Monitoring > lab tests
   - Track conversions, not just traffic

9. **Iterative Improvement**
   - Regular updates beat one-time perfection
   - Monitor, measure, optimize continuously
   - Small improvements compound over time

10. **Speed + Stability + Accessibility = Success**
    - Fast loading (LCP < 2.5s)
    - Stable layout (CLS < 0.1)
    - Accessible to all (WCAG 2.2 AA)
    - These three pillars = great user experience
